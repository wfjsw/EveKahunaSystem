# 工业
让我们直入主题，本站的核心功能：工业规划。

你可以通过左侧的菜单快捷的进入工业界面。

## 界面介绍
![系统工业辅助流程示意图](/assets/doc-industry-1.png)
左侧是市场树，用来选择需要加入计划的物品，右键菜单可以选择加入计划和查看物品详细信息。

中间是当前计划中的安排产品，可以拖拽调整顺序，可以改变数量。需要点击保存才能存储顺序。

右侧是当前计划的详细配置，即配置流，用来告知计划中的执行细节，如
- 建筑插件
- 蓝图系数
- 使用库存
- 最大流程数量
- ......

配置流按照顺序从上到下以此生效，详细请见配置流介绍。

## 配置流

配置有6种：
- 建筑分配：定义制造环节被分配到哪个建筑，会影响建筑插件的参数获取、物流环节的库存计算
- 建筑插件：影响建筑插件的参数获取
- 原材料标记：被标记为原材料的可生产物品不会再向下分解。没有蓝图的物品默认为原材料。本平台暂时不会考虑行星产物的分解。
- 缺省蓝图参数：当开启蓝图库存检测时，无法实际分配的蓝图以及没有开启蓝图库存检测时所有产品使用的蓝图系数，可以添加多个按顺序进行匹配筛选。
- 载入库存：alpha订阅功能，载入在库存管理界面设定的监控库存容器
- 最大作业拆分：控制某种产品的最大单线流程，可以按照流程数或大概时间进行限制。

### 如何生效？
以建筑分配为例
![建筑分配图例](/assets/doc-industry-2.png)
一个工作进入建筑分配流程后会从上到下开始匹配，舰船会在第一个配置被分配到sotiyo，而反应则直到第三个配置被分配到tatara。
配置和计划一样可以拖动调整顺序，调整后需要保存。

通过多样的配置和顺序定义你自己的计划模型。

# 报表
![报表图例](/assets/doc-industry-3.png)

在修改计划配置后需要点击 立刻计算 按钮获取计算结果。
计算结果获取后保存在本地，可随时查看。

报表丰富多样，请自行探索用法。
